Bitmap的高效加载

BitmapFactory类提供了四类方法：decodeFile、decodeResource、decodeStream和decodeByteArray来加载出一个bitmap对象，其中decodeFile和decodeResource又间接调用了decodeStream方法。

通过bitmapFactory.Options可以按一定的采样率inSampleSize来加载缩小后的图片。inSampleSize的取值应该总是为2的指数。获取采样率流程：
1.将bitmapFactory.Options的inJustDecodeBounds参数设置为true并加载图片。
2.从bitmapFactory.Options中取出图片的原始宽高数据。
3.根据采样率的规则并结合目标view的大小计算出采样率。
4.将bitmapFactory.Options的inJustDecodeBounds参数设置为false，然后重新加载图片。

Android中的缓存策略

最常用的缓存算法是LRU，核心是当缓存满时，会优先淘汰那些近期最少使用的缓存对象，系统中采用LRU算法的缓存有两种：LruCache(内存缓存)和DiskLruCache(磁盘缓存)。

LruCache
LruCache是Android 3.1才有的，通过support-v4兼容包可以兼容到早期的Android版本，也建议使用support-v4兼容包提供的LruCache。LruCache是一个泛型类，它内部采用一个linkedHashMap以强引用的方式储存外界的缓存对象。强引用:直接的对象引用。软引用:系统内存不足时此对象会被gc回收。弱引用:此对象会随时被gc回收。LruCache是线程安全的。

int maxMemory = (int) (Runtime.getRuntime().maxMemory() / 1024);
    int cacheSize = maxMemory / 8;
    mMemoryCache = new LruCache<String, Bitmap>(cacheSize) {
        @Override
        protected int sizeOf(String key, Bitmap bitmap) {
            return bitmap.getRowBytes() * bitmap.getHeight() / 1024;
        }
    };

mMemoryCache.get(key);
mMemoryCache.put(key,bitmap);

DiskLruCache
DiskLruCache不属于AndroidSDK的一部分，获取的源码不能直接在Android中使用，需要稍微修改编译错误。

1.创建。不能通过构造方法创建，需要用open方法。

public static DiskLruCache(File directory,int appVertion,int valueCount,long maxSize)

第一个参数表示缓存文件在系统中的存储路径。第二个参数表示应用的版本号，一般设为1即可。第三个参数表示单个节点所对应的个数，一般设为1即可。第四个参数表示缓存的总大小。

private static final long DISK_CACHE_SIZE = 1024 * 1024 * 50;
File diskCacheDir = getDiskCacheDir(mContext, "bitmap");
if (!diskCacheDir.exists()) {
    diskCacheDir.mkdirs();
}
mDiskLruCache = DiskLruCache.open(diskCacheDir, 1, 1,
                    DISK_CACHE_SIZE);

2.添加。添加操作是通过Editor完成的，通过editor()获取Editor对象，如果这个缓存正在被编辑那么会返回null。一般会采用url的md5值作为key，防止url中的特殊字符影响在Android中使用。

String key = hashKeyFormUrl(url);
DiskLruCache.Editor editor = mDiskLruCache.edit(key);
OutputStream outputStream = editor.newOutputStream(DISK_CACHE_INDEX);
if (downloadUrlToStream(url, outputStream)) {
    editor.commit();
} else {
    editor.abort();
}
mDiskLruCache.flush();

3.查找。先将url转换成key，然后通过get方法获得一个Snapshot对象，接着可以通过Snapshot对象获取缓存文件的输入流，然后获取bitmap对象。需先用bitmapFactory.Options对图片进行处理，通过BitmapFactory.decodeFileDescriptor而不是BitmapFactory.decodeResource。

Bitmap bitmap = null;
String key = hashKeyFormUrl(url);
DiskLruCache.Snapshot snapShot = mDiskLruCache.get(key);
if (snapShot != null) {
    FileInputStream fileInputStream = (FileInputStream)snapShot.getInputStream(DISK_CACHE_INDEX);
    FileDescriptor fileDescriptor = fileInputStream.getFD();
    bitmap = mImageResizer.decodeSampledBitmapFromFileDescriptor(fileDescriptor,reqWidth, reqHeight);
    if (bitmap != null) {
        addBitmapToMemoryCache(key, bitmap);
    }
}

优化列表的卡顿现象
(1)不要在getView中执行耗时操作，不要在getView中直接加载图片，否则肯定会导致卡顿,应用异步的方式；
(2)控制异步任务的执行频率：在列表滑动的时候停止加载图片，等列表停下来以后再加载图片；

@Override
public void onScrollStateChanged(AbsListView view, int scrollState) {
    if (scrollState == OnScrollListener.SCROLL_STATE_IDLE) {
        mIsGridViewIdle = true;
        mImageAdapter.notifyDataSetChanged();
    } else {
        mIsGridViewIdle = false;
    }
}

if (mIsGridViewIdle && mCanGetBitmapFromNetWork) {
    imageView.setTag(uri);
